## 🚨 トラブル対応

### サービス停止時
```bash
calmie-server
cd ~/hitoikiAPI/services/calmie
./health-check.sh  # 状態確認
./start.sh prod     # 再起動
```

### ディスク容量不足
```bash
calmie-cleanup  # 自動クリーンアップ
```

### 緊急ロールバック
```bash
cd ~/hitoikiAPI/hitoikiAPI
git log --oneline -5  # 最近のコミット確認
git revert HEAD       # 直前の変更を取り消し
calmie-deploy        # デプロイ
```

## 💡 開発のコツ

### 効率的な開発
- ホットリロードを活用
- pgAdmin でDB操作を可視化  
- リアルタイムログでデバッグ

### 安全なデプロイ
- 必ずローカルテスト完了後にデプロイ
- 小さな変更を頻繁にデプロイ
- 緊急時のロールバック準備

### チーム開発
- コミットメッセージは具体的に
- 大きな変更は事前相談
- 本番デプロイ時は他メンバーに通知
